# 数据集视频抽帧(vdo.avi → 图片序列)

在线 MTMCT 主流程 [`2. online_MTMC/run_mtmc.py`](2.%20online_MTMC/run_mtmc.py) 并不是直接读取视频,
而是**逐帧读取图片文件**。它按如下路径查找每个相机的帧图像:

```python
img_dir = os.path.join(opt.data_dir, cam) + '/frame/*'
```

因此在运行跟踪之前,必须先把 AIC19 数据集里每个相机的 `vdo.avi` 抽帧成图片。
本仓库已经提供了可直接使用的抽帧脚本:
[`1. train_feat_ext/utils/osUtils.py`](1.%20train_feat_ext/utils/osUtils.py)。

---

## 目录结构

**抽帧前**(AIC19 原始数据,每个相机目录下有 `vdo.avi`):

```
dataset/AIC19/validation/S02
├── c006
│   ├── vdo.avi
│   ├── roi.jpg
│   └── ...
├── c007
├── c008
└── c009
```

**抽帧后**(每个相机多出一个 `frame/` 目录):

```
dataset/AIC19/validation/S02
├── c006
│   ├── vdo.avi
│   └── frame
│       ├── c006_f0001.jpg
│       ├── c006_f0002.jpg
│       └── ...
├── c007
│   └── frame/c007_f0001.jpg ...
└── ...
```

> **命名约定**:`<相机ID>_f<帧号>.jpg`,帧号从 **1** 开始、补零到 **4 位**(如 `c006_f0001.jpg`)。
> 这是主流程默认识别的命名方式,请不要随意更改。

---

## 脚本做了什么

[`1. train_feat_ext/utils/osUtils.py`](1.%20train_feat_ext/utils/osUtils.py) 包含两个函数:

- `get_video_list(root_dir)`:递归遍历给定目录,收集所有视频文件(`.mp4/.avi/.mov/.mkv/.wmv`)。
- `video_to_frames(video_path, output_folder, camera_id)`:用 OpenCV 逐帧读取视频,
  把每一帧写成 `{camera_id}_f{帧号:04d}.jpg` 存入该相机的 `frame/` 目录。

`__main__` 部分会自动完成"遍历 S02 → 逐个相机抽帧"的整套流程:

```python
if __name__ == '__main__':
    mtmct_dataset = '.../dataset/AIC19/validation/S02'   # 数据集根目录
    video_list = get_video_list(mtmct_dataset)           # 找到所有 vdo.avi
    for video_path in video_list:
        camera_id = video_path.split('/')[-2]            # 从路径中取出相机ID,如 c006
        video_to_frames(video_path,
                        f'../../dataset/AIC19/validation/S02/{camera_id}/frame',
                        camera_id)
```

---

## 使用方法

1. 打开 [`1. train_feat_ext/utils/osUtils.py`](1.%20train_feat_ext/utils/osUtils.py),
   把 `__main__` 里的两处路径改成你本机的实际路径:
   - `mtmct_dataset`:AIC19 的 `S02` 数据集**绝对路径**;
   - `video_to_frames(...)` 的第二个参数(输出目录)——确保它指向 `.../S02/{camera_id}/frame`。

2. 在 `1. train_feat_ext/utils/` 目录下运行脚本:

   ```bash
   cd "1. train_feat_ext/utils"
   python osUtils.py
   ```

3. 运行结束后,每个相机目录下都会生成 `frame/` 文件夹及其中的图片序列,
   即可回到 `2. online_MTMC/` 执行 `python run_mtmc.py`。

---

## 注意事项

- **工作目录**:脚本里输出目录用的是相对路径 `../../dataset/...`,是相对于**脚本运行时所在目录**解析的。
  建议按上面的方式 `cd` 到 `1. train_feat_ext/utils/` 再运行;或直接把输出目录改成绝对路径,避免帧被写到意外位置。
- **相机ID的提取**:`video_path.split('/')[-2]` 依赖 `.../c006/vdo.avi` 这种"相机目录 + vdo.avi"的层级结构。
  Windows 下若路径用反斜杠 `\`,可改用 `os.path.basename(os.path.dirname(video_path))` 更稳妥。
- **帧号从 1 开始**:脚本里用的是 `frame_count + 1`,与主流程的 1 起始约定一致,切勿改成从 0 开始。
- **换数据/场景**:要抽其它 scene(如 `S01`)或测试集的帧,只需把 `mtmct_dataset` 与输出目录里的
  `validation/S02` 换成对应的 split / scene 即可,脚本会自动处理该目录下的所有相机。
- **磁盘占用**:抽帧会把整段视频展开成大量 JPG,单个 scene 通常有数千帧 × 多相机,请预留足够磁盘空间。
